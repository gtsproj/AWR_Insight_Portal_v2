## ARCHITECTURE & KEY DECISIONS

### Trigger Flow (as of 2026-06-24)
```
AWR file parsed by master_parser.py
  └─ All 48 AWR modules run (SAR + plan excluded)
  └─ 7 Materialized Views refreshed (CONCURRENT)
  └─ _get_snap_range() resolves begin_snap/end_snap from awr_load_profile
  └─ recommendation_engine.run() — stores to awr_recommendations
  └─ anomaly_detector.detect() per snap — stores to awr_anomalies (AWR metrics only)
  └─ Archive to archive/<dbname>/

SAR file parsed by sar_master_parser.py
  └─ 10 SAR parsers run
  └─ _resolve_sar_time_range() — queries sar_cpu_stats for recent rows
  └─ anomaly_detector.detect_sar() — stores to awr_anomalies (SAR metrics only)
```

### Comparison Dashboard Flow
```
Portal /comparison → Create baseline + current tags (DB + Instance + Snap)
  └─ "Open Comparison Dashboard" button → Grafana with ?var-baseline_tag=X&var-current_tag=Y
       └─ Dashboard reads snap range from awr_comparison_tags
       └─ No DB/instance dropdowns — everything scoped via tag JOIN
```

### Interface Detection Order (master_parser → sub-engines)
- recommendation_engine: run() → generate_recommendations() → main() with sys.argv patch
- anomaly_detector: detect() → run() → main() with sys.argv patch

### PostgreSQL Transaction Safety
- After a failed query, all subsequent queries on the same connection abort
- Every independent query wrapped in try/except + conn.rollback()

### SAR Timestamp Handling
- Binary SAR files: TZ=Asia/Kolkata sadf -c + sar -A via WSL → correct IST timestamps
- hostname_override parameter prevents WSL machine name appearing as hostname

## NOTES

### SAR Drop Folder Convention
Mirror of AWR reports folder structure. Pre-create one subfolder per server:
```
sar_drop\
  TCLFSLPRDDB1\    ← drop sa01, sa02, sa03.txt here
  TCLFSLPRDDB2\    ← add new servers by creating new subfolder
```
Watcher uses subfolder name as hostname. No renaming of sa01/sa02 files needed.
Binary files (sa01, sa02) auto-converted via WSL sadf -c if WSL is available.
Text files (.txt) queued directly. Both result in sar_queues\queue_<HOSTNAME>.json.
Archive: sar_archive\<HOSTNAME>\ after queuing.

### SAR Binary File Conversion
Binary SAR files (sa01, sa02...) from Oracle Linux sysstat 10.1.5 need
conversion before parsing. The portal handles this automatically via WSL:
  wsl bash -c "TZ=Asia/Kolkata sadf -c <file> > /tmp/conv && TZ=Asia/Kolkata sar -A -f /tmp/conv"
TZ=Asia/Kolkata is required so output times are IST not UTC.
If WSL is unavailable: sar -A -f <file> > output.txt on the Linux server.

### HugePages Interpretation
hugused_pct near 100% is NORMAL for Oracle (SGA pre-allocates all HugePages).
Alert condition: kbhugfree approaching 0 — Oracle cannot grow SGA further.
Sudden drop in kbhugused = Oracle instance crashed or SGA shrunk.

### AWR Comparison Workflow
1. Portal → Comparison page → select DB + Instance
2. Load snaps → select start/end snap → create baseline tag
3. Create current tag for the after-fix snap range
4. Click "Open Comparison Dashboard" — opens Grafana pre-loaded with both tags
5. Dashboard shows Period Info table + Wait Events + SQL + Load Profile deltas
