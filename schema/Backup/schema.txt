There is a issue in the master_parser.py and the modules script created.
Issues observed are
1. Data is either incorrectly inserted (column mismatch) or is not inserted at all
2. If any section in the AWR  is empty / no records then the parser script instead of skipping the section, it
populates the data of the next section.

We need to revisit the master_parser.py and all the other module script o rectify these errors. Secondly I propose to created individual output files for each section. Need to parse only the columns mentioned in the table structure below and in the same order. This will help to easily map the table columns with the columns in the parsed output file. If possible records should be populated in the table during the parsing stage rather then executing it as a separate script. This will ensure continuity in the modules.

The master_parser.py and the module parsers are working fine except for the missing section issue.
